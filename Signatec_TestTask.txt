Тестовое задание С++

Постановка задачи:
Необходимо реализовать консольное приложение - диспетчер процессов (аналог утилиты top). Приложение мониторит запущенные в ОС процессы и выводит их в cout. Список имён процессов для мониторинга программа получает через аргументы командной строки. Мониторинг каждого отдельного процесса должен происходить в отдельном потоке. Обновление информации об отслеживаемом процессе должно происходить по одному из двух событий: - Изменение потребления CPU больше чем на 5% - Изменение потребления памяти больше чем на 5 МБ
Сборка через CMake.
 
Юнит тесты будут плюсом. 

Технические требования: 
GCC >= 10.0 CMake >= 3.15 
Версии стандарта C++ от 11 до 17. 
Разрешается использовать только stl и boost.
